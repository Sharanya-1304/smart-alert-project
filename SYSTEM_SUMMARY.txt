================================================================================
                         HOW CAREGIVER ALERTS WORK
================================================================================

FIRE DETECTION:
   → System detects orange-red colored pixels in video
   → Gets CAREGIVER EMAIL from user's profile
   → Sends FIRE ALERT email with subject "FIRE EMERGENCY"
   → Plays voice message: "Fire Alert! Fire has been detected!"
   → Sends SMS: "FIRE ALERT: Fire detected. Evacuate!"
   → Triggers fire alarm sound
   → Logs alert in database

FALL DETECTION:
   → System detects person's body angle > 20 degrees
   → Gets CAREGIVER EMAIL from user's profile
   → Sends FALL ALERT email with subject "FALL DETECTED"
   → Plays voice message: "Fall Alert! A person has fallen!"
   → Sends SMS: "FALL ALERT: Person has fallen!"
   → Triggers fall alarm sound
   → Logs alert in database

BOTH FIRE AND FALL:
   → System detects both threats simultaneously
   → Sends CRITICAL EMERGENCY alert
   → Highest priority notifications
   → Emergency level email, voice, SMS, and sound

================================================================================
                       DATABASE INTEGRATION
================================================================================

USER TABLE stores:
   - user_id
   - username
   - email
   - caregiver_email <-- NEW FIELD FOR YOUR CAREGIVER

ALERT TABLE stores:
   - alert_id
   - user_id (monitored person)
   - alert_type (fire, fall, or fire_and_fall)
   - confidence (92%, 90%, etc.)
   - created_at (timestamp)
   - is_resolved (yes/no)
   - description
   - video_path

When detection happens:
   1. Get user's caregiver_email from USER table
   2. Create record in ALERT table
   3. Call trigger_alert(type, caregiver_email)
   4. Alert sent to caregiver immediately
